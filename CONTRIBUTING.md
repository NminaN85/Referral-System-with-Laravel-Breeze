# Contributing

Thanks for your interest — contributions and feedback are welcome!

Getting started
- Fork the repo and open a pull request.
- For quick testing, run:
  - Install: `npm install` (or your project's install command)
  - Run: `npm start` or `./run.sh`
- See `README.md` for full setup and examples.

How to give feedback
- Use the Discussions tab and open a thread under "Feedback", or:
- Open an issue using the "Feedback" or "Bug report" templates.

How to contribute code
1. Pick an issue labeled `good-first-issue` or `help-wanted`.
2. Create a branch: `git checkout -b fix/short-description`
3. Make changes, run tests, and commit.
4. Open a pull request with a clear title and description.
5. Reference related issues with `Fixes #<issue-number>` if applicable.

Code guidelines
- Keep changes small and focused.
- Follow existing coding style.
- Add tests for new features when possible.

Communication
- Be respectful and inclusive — please follow the project's Code of Conduct.
- If you're unsure about a change, open an issue first describing your idea.

Thank you — we appreciate your time and feedback!
